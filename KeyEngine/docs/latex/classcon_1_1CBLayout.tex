\doxysubsection{Detailed Description}
the layout class serves as a shell to hold the root of the \doxylink{classcon_1_1CBElement}{CBElement} tree 

=============================================================

\begin{DoxyAuthor}{Author}
Key\+C0de 
\end{DoxyAuthor}
\begin{DoxyDate}{Date}
2022/08/21 19\+:56
\end{DoxyDate}
client does not create CBElements directly, create a raw layout and then

use it to access the elements and add on from there. When building is done,

raw layout is moved to \doxylink{classBindableRegistry}{Bindable\+Registry} (usually via Buffer\+::make), and the internal layout

element tree is "{}delivered"{} (finalized and moved out). \doxylink{classBindableRegistry}{Bindable\+Registry} returns a baked

layout, which the buffer can then use to initialize itself. Baked layout can

also be used to directly init multiple Buffers. Baked layouts are conceptually immutable. \doxysubsection*{Base \doxylink{classcon_1_1CBLayout}{CBLayout} class cannot be constructed. }
